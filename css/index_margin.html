block vs inline

block level - 한줄을 통으로 차지
inline - 한줄안에 들어감
어떤건 block 이 합리적, 어떤건 inline이 합리적
이미 있는 걸 바꾸려면
style="display:inline" or display:block

box model

padding - 테두리 안에서 여백

margin - 테두리 밖에서 여백

Inline element의 경우 width / height 가 작동하지 않음.

box size

Hello this is small
hi this is large

width를 생각하면, border를 포함한 영역이라고 생각하기 쉬우나,
옛날 css가 나왔던 시절에는 padding, margin등의 영역이 없었다.
그래서, 오로지 content size == width였다.
이것 때문에 border 를 포함한 사이즈가 예측이 어려워짐

solution : Box sizing!

box-sizing:content-box (CSS default)
box-sizing:border-box (padding, border 포함. margin 불포함)

Margin Collapsing

2 element 사이의 간격은, 둘 사이의 더 큰 마진으로 이루어진다.
2 element의 마진을 더한 값이 적용되지 않음
장점 : li-tag 같은 경우 마진겹침 현상이 있으면 더 수월

부모 자식간의 margin collapsing

부모가 어떤 시각적 효과가 없을 때(border 같은 것도 없을 때)
부모와 자식간의 margin 은 서로 겹친다. 효과가 있으면 더해짐
margin의 개념을 ** 이 요소를 다른 보이는 인접한 요소와 떨어뜨린다 **라고 생각.

어떤 element 가 시각적인 효과가 전혀 없을 때,
그 element 의 margin은 max(top, bottom)이다. Not top + bottom
